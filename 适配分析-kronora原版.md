# 《甲方模拟器》原版游戏分析与 v2.0 适配方案

> **原版**：[kronora-delmaxmax.github.io/jiafang-simulator](https://kronora-delmaxmax.github.io/jiafang-simulator/)（已阅读完整源码）
> **目标**：用《甲方模拟器·升职记》v1.0/v1.1 剧情替换原版内容
> **核心问题**：原版是「乙方单线剧情」，我们是「多线升职系统」——改造不是简单换皮，是**架构重设计**

---

## 一、原版深度分析（先吃透再改）

### 1.1 原版的成功之处（要保留）

| 元素 | 表现 | 我们的剧情里能完美对应 |
|------|------|---------------------|
| **微信聊天界面** | 沉浸式 UI，emoji、语音条、打字指示器 | ✅ 完全兼容 |
| **左滑/右滑决策** | Tinder 式决策 + 键盘快捷键 | ✅ 完美适配 |
| **压力条** | 实时变化，绿/黄/红三色 | ✅ 直接用 |
| **真人故事** | 100% 取材于乙方真实段子 | ✅ 我们的素材更丰富 |
| **5 个结局** | 不同价值观的归宿 | ✅ 我们 12 个结局更全 |
| **分享功能** | navigator.share + 复制文案 | ✅ 复用 |
| **重启游戏** | 一键重玩 | ✅ 复用 |
| **音效/震动** | CSS 动画模拟（粒子、横幅、shake） | ✅ 复用 |
| **进度反馈** | 微信通知横幅 + 闹钟提醒 | ✅ 复用 |

### 1.2 原版的核心局限（我们能超越）

| 局限 | 说明 | 我们的解法 |
|------|------|-----------|
| **单线剧情** | 一条路走到底，4 个场景 | **v1.0 7 幕 + v1.1 30+ 支线** |
| **单角色** | 只能演"小李"（乙方） | **4 个出身可选**（程序员/设计师/HR/原版小李） |
| **5 个结局** | 结局偏少 | **12 个结局**（4 隐藏） |
| **2 个分支** | 每个场景左右 2 选 1 | **30+ 分支** |
| **无朋友圈** | 缺一个关键反馈机制 | **朋友圈动态系统** |
| **无人物关系** | 王总就是王总，没同事 | **12 个 NPC 关系网** |
| **无乙里乙气系统** | 缺文化梗 | **乙里乙气等级 L0-L8** |

## 二、改造的 3 种方案（从轻到重）

### 方案 A：最小改造（3-5 天）—— 简单换皮

**做法**：保持原版架构和 4 个场景，**只把对话和决策换成我们的剧情**

**优点**：
- 上线最快
- 风险最低
- 保留原版所有技术红利

**缺点**：
- 我们 v1.0 的多线剧情没法展开
- 朋友圈、人物关系、乙里乙气系统都用不上
- 只是"原版 2.0"不是真正的"升职记"

**适用场景**：1 周内要上线 MVP 验证

### 方案 B：中等改造（2-3 周）—— 架构升级

**做法**：保留原版 UI 框架，**重写游戏引擎**，支持多线剧情 + 朋友圈 + 人物关系

**核心改造**：
1. 引入「场景图」数据结构（节点 + 边）
2. 引入「全局状态」（5 大属性 + 10 个 NPC 关系）
3. 引入「朋友圈组件」（右侧弹层）
4. 引入「乙里乙气等级」系统
5. 增加「支线触发器」

**优点**：
- 充分利用我们的剧本
- 保留沉浸式聊天体验
- 实现真正"模拟器"感

**缺点**：
- 需要 1 个前端工程师
- 工作量适中
- 朋友圈 UI 要设计

**适用场景**：1 个月内要上线，团队有前端支持

### 方案 C：深度改造（4-6 周）—— 完整重做

**做法**：保留"甲方模拟器"品牌 + 核心 UI 模式，**重新设计游戏为多线剧情 + 关系网 + 朋友圈 + 多周目**

**核心改造**：
1. 重做数据层（剧情图 + NPC 关系 + 状态机）
2. 重做 UI（增加朋友圈面板 / NPC 通讯录 / 状态仪表盘）
3. 重做结局系统（12 个结局 + 隐藏条件）
4. 增加「时间跳跃」机制（6 个月压缩成 90 分钟）
5. 增加「周目切换」（第二周目反派视角）
6. 增加「乙里乙气」独立系统

**优点**：
- 真正还原"模拟器"感
- v1.0/v1.1 全部素材都用上
- 商业化潜力大（v2.0/v3.0 路线都通）

**缺点**：
- 工作量大
- 需要完整前端团队
- 4-6 周交付

**适用场景**：3 个月内要上线，有完整团队

#### 3.1.2 UI 升级

**新增面板**：

1. **朋友圈面板**（右侧滑出）
   - 玩家做的每个关键选择后，弹出 1-2 条朋友圈
   - 玩家可以点赞/评论（fake 评论，不影响剧情）
   - 朋友离开页面，朋友圈成为"回忆"

2. **NPC 通讯录**（底部 tab）
   - 12 个 NPC 头像
   - 点开看和当前 NPC 的关系值
   - "信任度 < 30"显示警告图标

3. **属性仪表盘**（顶部新增）
   - 5 大属性 + 抗压值
   - 不用一直显示，按需展开
   - 关键抉择前自动弹出预览

4. **时间跳跃**（核心创新）
   - 一周（5 个工作日）压缩成 5 个快速场景
   - 每周结束有一次"周末总结"
   - 朋友圈刷一波

#### 3.1.3 引擎升级

**核心函数**：
```js
// 节点跳转引擎
function goToNode(nodeId) {
  const node = StoryGraph.nodes[nodeId];
  
  // 触发节点事件
  for (const event of node.events || []) {
    applyEvent(event);
  }
  
  // 播放对话
  await runScene(node.messages, node.decision);
  
  // 触发朋友圈
  if (node.moments) {
    await showMoments(node.moments);
  }
  
  // 进入下一节点
  if (node.next) {
    await goToNode(node.next);
  }
  
  // 结局
  if (node.ending) {
    showEnding(node.ending);
  }
}

// 事件处理器
function applyEvent(event) {
  if (event.type === "stress") setStress(state.stress + event.delta);
  if (event.type === "business") state.business += event.delta;
  if (event.type === "trust") state.npc[event.npc].trust += event.delta;
  if (event.type === "moment") addMoment(event.content);
  if (event.type === "flag") state.flags[event.flag] = true;
  if (event.type === "item") state.inventory.push(event.item);
}
```

## 七、关键技术决策

### 7.1 框架选择

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| 纯原生 HTML/CSS/JS | 跟原版一致，零依赖 | 代码组织差 | ⭐⭐⭐ |
| Vue 3 + Vite | 轻量、好组织 | 需要构建 | ⭐⭐⭐⭐ |
| React + Vite | 生态好 | 重 | ⭐⭐ |
| Svelte | 最轻 | 学习成本 | ⭐⭐⭐⭐ |

**推荐：Vue 3 + Vite**
- 跟原版一样单文件
- 但能用组件化组织
- 状态管理简单

### 7.2 状态管理

| 方案 | 适用 |
|------|------|
| Pinia | 复杂状态、调试好 |
| Vue reactive() | 简单状态、够用 |
| 原生 JS 对象 | 跟原版一致 |

**推荐：Pinia**
- 状态变更可追踪
- 朋友圈/属性/NPC 关系都需要响应式

### 7.3 部署

- **GitHub Pages**（跟原版一致）
- **Vercel / Netlify**（推荐，自动部署）
- **微信 H5**（朋友圈传播）
